3. INSTALLATION COMMANDS
================================================================================

STEP 1: Clone or Pull the Repository
================================================================================

First time setup (clone from GitHub):
```
git clone https://github.com/MLNdlovu/VOICE-ASSISSTANT-CALENDAR.git
cd VOICE-ASSISSTANT-CALENDAR
```

OR if already have repo, pull latest changes:
```
cd VOICE-ASSISSTANT-CALENDAR
git pull origin main
```


STEP 2: Create Python Virtual Environment
================================================================================

Windows (PowerShell):
```
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Windows (Command Prompt):
```
python -m venv .venv
.venv\Scripts\activate.bat
```

macOS/Linux:
```
python3 -m venv .venv
source .venv/bin/activate
```

Verify activation - you should see (.venv) in your prompt.


STEP 3: Install All Dependencies
================================================================================

```
pip install -r requirements-voice.txt
```

This installs:
- Google Calendar API (google-api-python-client, google-auth-oauthlib)
- Voice: SpeechRecognition, PyAudio, pyttsx3
- GUI: tkcalendar
- Web: Flask
- AI: openai, python-dotenv
- Testing: pytest, pytest-cov
- Other: python-dateutil, prettytable


STEP 4: Set Up Google OAuth Credentials
================================================================================

1. Go to: https://console.cloud.google.com
2. Create a new project (name it "Voice Assistant Calendar")
3. Enable Google Calendar API:
   - Click "Enable APIs and Services"
   - Search for "Google Calendar API"
   - Click Enable
4. Create OAuth 2.0 Credentials:
   - Go to "Credentials"
   - Click "Create Credentials" ‚Üí "OAuth client ID"
   - Choose "Desktop application"
   - Click Create
   - Download JSON file
5. Save to project:
   - Create folder: mkdir .config
   - Move JSON file to: .config/client_secret_[YOUR_ID].apps.googleusercontent.com.json

The app will auto-discover the client secret file.


STEP 5: Optional - Set Up ChatGPT (for AI features)
================================================================================

1. Get OpenAI API key: https://platform.openai.com/api-keys
2. Create .env file in project root:
   ```
   OPENAI_API_KEY=sk-xxxxxxxxxxxxx
   ```
3. The app will auto-load this for ChatGPT features

8. TROUBLESHOOTING
================================================================================

ERROR: "ModuleNotFoundError: No module named 'X'"
Solution:
```
pip install -r requirements-voice.txt
```


ERROR: "Port 5000 already in use"
Solution:
```
python web_app.py --port 5001
```


ERROR: "Client secret not found"
Solution:
- Verify .config/client_secret_*.json exists
- Check file is in correct folder


ERROR: "PyAudio installation fails"
Solution:
- You're using Python 3.13+ (wrong!)
- Install Python 3.11.x instead
- Create new venv with Python 3.11


ERROR: "Microphone not working"
Solution:
```
pip install --upgrade pyaudio
```


ERROR: "OAuth login fails"
Solution:
- Clear browser cookies
- Delete token files: rm .config/token_*.json
- Try signing in again


ERROR: "Permission denied" (macOS/Linux)
Solution:
```
chmod +x voice_assistant_calendar.py
chmod +x web_app.py
```


ERROR: "git push rejected"
Solution:
```
git pull origin main
git push origin main
```

================================================================================
SECURITY NOTES
================================================================================

‚ö†Ô∏è IMPORTANT - DO NOT COMMIT:

1. .config/ folder (Google OAuth credentials)
   - These are tied to your Google Cloud project
   - Anyone with these files can access your calendar
   - Store locally only, not in git

2. .env file (API keys)
   - Contains OPENAI_API_KEY
   - Anyone with this can use your API quota
   - Store locally only, not in git

3. Token files (.config/token*.json)
   - Contains access tokens
   - Can be regenerated by signing in again
   - Store locally only, not in git

BEST PRACTICE:
1. Add .gitignore to exclude these files
2. Include .env.example as template
3. Document setup process in README
4. Inform team members of security requirements
