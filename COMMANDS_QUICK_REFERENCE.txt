COMPUTER 1 (NOW) - PUSH TO GITHUB
================================================================================

1. Check what changed:
   git status

2. Add everything:
   git add .

3. Commit your changes:
   git commit -m "Add web dashboard, Flask server, and complete documentation"

4. Push to GitHub:
   git push origin main

5. Verify it worked:
   git log --oneline -1


COMPUTER 2 (TOMORROW) - PULL FROM GITHUB
================================================================================

1. Install Python 3.11+ first!
   https://www.python.org/downloads/
   (Use 3.11, NOT 3.13+)

2. Clone the repository:
   git clone https://github.com/MLNdlovu/VOICE-ASSISSTANT-CALENDAR.git
   cd VOICE-ASSISSTANT-CALENDAR

3. Create virtual environment:
   python -m venv .venv

4. Activate it (Windows PowerShell):
   .\.venv\Scripts\Activate.ps1

   OR (Windows Command Prompt):
   .venv\Scripts\activate.bat

   OR (macOS/Linux):
   source .venv/bin/activate

5. Install all packages:
   pip install -r requirements-voice.txt

6. Copy Google credentials:
   - Copy .config/ folder from Computer 1 to Computer 2
   - Paste in project root

7. Verify setup:
   pytest

   Expected: "50 passed"

8. Run the app (choose one):
   
   GUI:
   python voice_assistant_calendar.py
   
   Web:
   python web_app.py
   Then open: http://localhost:5000


EVERY TIME YOU OPEN A NEW TERMINAL
================================================================================

Activate virtual environment:

Windows PowerShell:
.\.venv\Scripts\Activate.ps1

Windows Command Prompt:
.venv\Scripts\activate.bat

macOS/Linux:
source .venv/bin/activate

(You'll see (.venv) in your prompt when activated)

TROUBLESHOOTING QUICK FIXES
================================================================================

"ModuleNotFoundError: No module named X":
  pip install -r requirements-voice.txt

"PyAudio install fails":
  Use Python 3.11, not 3.13+

"Port 5000 already in use":
  python web_app.py --port 5001

"OAuth credentials not found":
  Copy .config/client_secret_*.json from Computer 1

"Tests fail":
  pytest -v  (shows which test failed)

"Git push fails":
  git pull origin main
  git push origin main

"Virtual environment not activating":
  Make sure you're in the project folder
  Check you're using correct activation command for your OS

HELPFUL ALIASES (Optional)
================================================================================

Create shortcuts for frequent commands:

Windows PowerShell (add to profile):
function venv { .\.venv\Scripts\Activate.ps1 }
function test { pytest }
function run-gui { python voice_assistant_calendar.py }
function run-web { python web_app.py }

Linux/Mac (.bashrc or .zshrc):
alias venv='source .venv/bin/activate'
alias test='pytest'
alias run-gui='python voice_assistant_calendar.py'
alias run-web='python web_app.py'

Then just type: venv, test, run-gui, etc.
